One CLI Masters Google Workspace APIs Dynamically for Devs and AI Agents

gws Builds Commands at Runtime from Google's Discovery Service, Unifying Drive, Gmail, Calendar, and More in Structured JSON

googleworkspace/cli Rust Latest: v0.4.4 8.1k stars

In the sprawling ecosystem of Google Workspace, developers have long wrestled with fragmented tools: separate SDKs for Drive, Gmail, Sheets, and beyond, each demanding custom boilerplate, OAuth hurdles, and endless API documentation dives. Enter gws, the Google Workspace CLI from googleworkspace/cli—a single, Rust-powered command-line powerhouse that dynamically generates its entire command surface from Google's own Discovery Service at runtime.

This is no static binary with hardcoded endpoints. When Google evolves its Workspace APIs—adding methods to Calendar or extending Admin console access—gws adapts automatically, pulling fresh schemas on-the-fly. Fire up gws drive files list --params '{"pageSize": 5}' and it spits structured JSON, complete with auto-pagination and --dry-run previews. Need --help on obscure resources? It's there, mirroring official docs without the browser tabs.

Installation is a breeze, sidestepping Rust toolchain woes via npm: npm install -g @googleworkspace/cli bundles pre-built binaries for macOS (Apple Silicon or Intel), Windows, and Linux. Nix flakes and Cargo work too, but npm handles the heavy lifting. Setup kicks off with gws auth setup, guiding OAuth credentials via a Google Cloud project, followed by gws auth login. No more curl-ing REST endpoints or scripting wrappers—gws is built for humans craving productivity.

But gws shines brightest for AI agents. Every output is crisp, parseable JSON, fueling 40+ baked-in "agent skills" like querying Chat threads or batch-editing Sheets. Pair it with tools like Gemini extensions or MCP servers for autonomous workflows. Developers report chaining it into pipelines: list Gmail drafts, filter via jq, pipe to automation scripts. It's not an official Google product—disclosure in the README—but its momentum underscores a real pain point solved.

Technically, Rust delivers snappy performance, with the CLI introspecting Discovery APIs to scaffold subcommands like gws sheets spreadsheets values append or gws admin reports activities list. Authentication scopes dynamically adjust per resource, minimizing privilege creep. Advanced users tap --params for raw JSON payloads, --fields for response pruning, and even light/dark terminal theme fixes in v0.4.4.

Why the buzz? gws obliterates silos. Instead of juggling gcloud for partial overlap or language-specific clients, one tool rules Workspace. Early adopters—admins scripting bulk operations, AI builders prototyping agents—praise its zero-boilerplate ethos. At just days old, it's exploding in developer circles, hinting at a v1.0 horizon amid active iteration.

For teams entrenched in Google Workspace, gws isn't just a CLI; it's a paradigm shift toward self-healing tools that evolve with the platform. Expect integrations galore as agentic workflows mature.

OpenClaw Backup Delivers One-Click Portability for AI Agent Instances

Shell scripts capture workspaces, credentials and histories, enabling frictionless restores to new servers without re-pairing channels.

LeoYeAI/openclaw-backup · Shell · 459 stars

Builders running OpenClaw instances—AI agents with full server access for coding, file management and scripting—now have a reliable way to safeguard and migrate their setups. The LeoYeAI/openclaw-backup project provides shell-based tools for comprehensive backups and restores, addressing a core pain point: the fragility of agent states across hardware changes or experiments.

OpenClaw, built on the MyClaw.ai platform, treats agents as persistent entities with workspaces, skills and session histories. Without proper backups, recreating these from scratch means reconfiguring API keys, bot tokens and channel pairings. This tool consolidates everything into a single tarball:

  • Workspace: MEMORY.md, skills, SOUL.md, USER.md and agent files.
  • Config: openclaw.json with model settings and tokens.
  • Credentials: Channel pairing states for automatic reconnection.
  • Sessions: Full conversation histories.
  • Cron jobs: Scheduled tasks.
  • Scripts: Guardian, watchdog and gateway starters.

Installation is straightforward. Agents can self-install via the command "Help me install backup", or builders run clawhub install myclaw-backup. Core scripts include backup.sh for archiving to a directory like /tmp/openclaw-backups, and restore.sh with a mandatory --dry-run flag for safety before full execution.

A browser UI simplifies workflows: serve.sh start --token $(openssl rand -hex 16) --port 7373 launches a local server at http://localhost:7373/?token=YOUR_TOKEN for download, upload and restore. Migration follows a clean sequence: backup and download on the old server, install OpenClaw and this skill on the new one, then upload and restore. Channels reconnect without intervention.

Version 1.7.0 introduces a post-restore recovery report. After restore.sh completes, it creates workspace/.restore-complete.json detailing the backup name, agent ID, timestamp and contents. On startup, the agent detects this flag, reads USER.md for language preference, notifies the user and deletes the file to avoid duplicates. This ensures builders confirm success immediately.

For developers iterating on AI agents, this eliminates downtime from server failures or cloud switches. It has quickly gained traction, with nearly 500 GitHub stars in days, signaling demand among OpenClaw users. Security demands caution: the tool handles sensitive credentials, so restrict access during backups.

Builders deploying persistent AI workflows should integrate this now—its zero-repairing restores make agent portability routine.

Cortex implements real-time PID control, DShot ESCs and nRF24 telemetry for custom drone builds

sergiovirahonda/cortex · C++ · 112 stars

Builders crafting DIY quadcopters now have Cortex, an open-source flight controller built around the ESP32-S3 microcontroller. Hosted at sergiovirahonda/cortex, it processes sensor data from an MPU6050 gyroscope and accelerometer to deliver real-time attitude stabilization via PD controllers for roll and pitch, plus yaw PI with rate feedforward.

The system communicates wirelessly via nRF24L01 modules, receiving low-latency commands from a paired Synapse transmitter based on joystick input. It drives 4-in-1 ESCs using native DShot600 protocol over ESP32's RMT hardware for precise motor control.

Key features include OLED telemetry displaying attitude, throttle and trims; radio downlink of flight states; altitude hold fusing LiDAR and accelerometer data with configurable engage/disengage; and heading lock blending compass and gyro inputs, falling back to gyro if needed.

Recent v1.1.0 release adds yaw integral term, Core0 tasking for parallel radio reads, and config injection. A detailed TUNING.md guides PID tuning, throttle bands and feature flags. At just over a month old, Cortex lowers barriers for affordable, tunable drone flight control using familiar C++ and ESP32 tools.
