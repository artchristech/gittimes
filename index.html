Python Backend Extends TradingView with Advanced Indicator Logic

Webhook server fetches live data to compute RSI, MACD and more, delivering composite signals to chat apps.

sharonubsyq/trading-view-indicator-extension Python Latest: 1.2 411 stars

TradingView's Pine Script offers powerful charting, but its limitations in handling complex, multi-timeframe indicator logic often force developers to seek external solutions. Enter trading-view-indicator-extension, a lightweight Python backend that bridges this gap. Launched six days ago—with over 400 stars on GitHub—it receives webhook alerts from TradingView, pulls fresh OHLCV data via Yahoo Finance, and runs an ensemble of indicators to generate actionable signals.

At its core, the project is a webhook server listening on /webhook for POST requests from TradingView alerts. Upon receipt, it validates the payload (optionally using an X-Webhook-Secret header for security), fetches recent candlestick data, and computes signals across five key indicators:

  • RSI (14-period): Detects bullish/bearish divergences across multiple timeframes.
  • MACD (12/26/9): Tracks signal line crosses, zero-line crosses, and histogram momentum.
  • Bollinger Bands: Monitors %B, bandwidth squeezes, and upper/lower breaks.
  • SuperTrend: Identifies direction flips with trend strength scoring for BUY/SELL.
  • VWAP: Anchored with 1σ/2σ bands, alerting on cross-ups or cross-downs.

These feed into a composite signal—a weighted score from -1.0 (STRONG SELL) to +1.0 (STRONG BUY)—returned as JSON. Developers can also query /signal/{symbol} directly for on-demand checks, bypassing TradingView alerts entirely.

What sets this apart is its notification pipeline. Signals route seamlessly to Telegram bots (with emoji ratings), Slack incoming webhooks, Discord, or custom Python callables. This turns TradingView into a full-fledged alert engine, ideal for automated trading workflows or monitoring dashboards.

Cross-platform support spans macOS, Linux, and Windows. A full pytest suite covers indicators and the server, while demo mode lets builders test without live data. The latest release, v1.2, refines these features—full changelog at the repo.

For quantitative developers or fintech builders, this solves the Pine Script ceiling: custom logic on live data without rebuilding charting tools. It's educational only—not financial advice, with clear risk disclaimers—but provides a robust, extensible foundation for research and prototyping algo strategies.

Builder's Take Review the disclaimer: this is for education, not live trading without due diligence on data sources like Yahoo Finance. Start with demo mode and `pytest` to validate indicators before integrating webhooks—secure them with secrets. Fork and extend the composite scoring logic to fit your strategy, as it's fully modular.

flamestro/deff · Rust · 366 stars

Developers reviewing git diffs now have deff, a Rust-based terminal user interface from flamestro. Launched last week, it provides side-by-side file comparisons with independent horizontal scrolling for each pane, vertical navigation, and line-level tinting for additions and deletions.

Key features include:

  • Per-file navigation with toggles (r key) that persist locally under .git.
  • Vim-like motions (h/j/k/l, g/G, Ctrl+u/d) plus mouse support, including shift-wheel for horizontal scroll.
  • In-diff search (/ then n/N) and language-aware syntax highlighting compiled into the binary.

Installation is straightforward: cargo install deff or via installer script (curl -fsSL https://raw.githubusercontent.com/flamestro/deff/main/install.sh | bash). Usage examples include deff --strategy range --base origin/main --head HEAD or deff --theme dark.

The v0.2.2 release adds cargo release automation. At six days old, deff addresses gaps in terminal diff tools, streamlining pre-commit inspections without leaving the CLI.

vphone-cli Boots Virtual iOS 26 iPhone on Apple Silicon Macs

Python tool exploits Virtualization.framework with research entitlements for full VM emulation

Lakr233/vphone-cli · Python · 2.8k stars

Lakr233's vphone-cli lets developers boot a complete virtual iPhone running iOS 26 on Apple Silicon Macs, tapping Apple's Virtualization.framework and PCC research VM infrastructure.

Tested on Mac16,12 hosts with macOS 26.3, it supports iPhone 17,3 models across iOS builds like 26.1-23B85 and 26.3-23D127, paired with compatible CloudOS versions. The workflow demands disabling SIP and AMFI via Recovery mode (csrutil disable; csrutil allow-research-guests enable) and setting nvram boot-args="amfi_get_out_of_my_way=1 -v".

Prerequisites include Homebrew packages (wget, gnu-tar, openssl@3, ldid-procursus, sshpass, keystone, build tools) and Git LFS for large IPSW archives. Key steps:

  • make setup_machine: Automates tools, trustcache, libimobiledevice, and Python venv (manual security config required).
  • make build: Compiles and signs the CLI.
  • make vm_new: Sets up VM directory with ROMs and storage.
  • make fw_prepare and make fw_patch: Extracts IPSWs, merges components, patches 41+ boot chain elements.

Restores use dual terminals: one for make boot_dfu (DFU mode), the other for interactive setup. This unlocks iOS VM testing without hardware, advancing macOS-hosted mobile development.

MuJoCo Delivers Fast, Accurate Physics for Robotics and Machine Learning Sims

Google DeepMind's engine simulates complex joint dynamics and contacts, powering research in biomechanics, graphics and AI control systems.

google-deepmind/mujoco C++ Latest: 3.5.0 12.2k stars

MuJoCo, short for Multi-Joint dynamics with Contact, is a physics engine designed for high-fidelity simulation of articulated structures in complex environments. Maintained by Google DeepMind since 2021, it targets researchers and developers in robotics, biomechanics, graphics, animation and machine learning. At its core, MuJoCo uses a C API optimized for speed, with a runtime module that preallocates low-level data structures via a built-in XML compiler. This setup minimizes overhead, enabling rapid iteration on simulations involving rigid bodies, tendons, actuators and realistic contacts.

What sets MuJoCo apart is its balance of accuracy and performance. It models continuous-time dynamics with discrete-time solvers, handling friction, collisions and soft constraints without sacrificing computational efficiency. The engine exposes utility functions for tasks like forward and inverse dynamics, energy computations and constraint Jacobians—essentials for control synthesis and optimization in robotics. Interactive visualization comes via a native OpenGL GUI in the simulate binary, allowing real-time model inspection and playback. For broader integration, Python bindings simplify scripting, while a Unity plug-in extends it to game engine workflows.

Recent version 3.5.0, detailed in the changelog, refines multithreading in the rollout module for parallel simulations and enhances the MJX differentiability layer for gradient-based learning in JAX. Builders tackling reinforcement learning agents or humanoid balancing will find the least-squares solver and LQR controller examples particularly useful.

Getting started is straightforward. Download and run simulate locally to interact with prebuilt models like a hopping humanoid or cartpole. Python users can dive into Google Colab notebooks covering basics, procedural model editing, rollouts, LQR synthesis—even balancing a humanoid on one leg via linear-quadratic control. Full documentation at mujoco.readthedocs.io includes XML references and API details.

For developers, MuJoCo's 12,000+ stars reflect steady adoption over 4.5 years, but its real value lies in enabling simulations too costly for real-world hardware. Robotics teams optimize trajectories without physical prototypes; ML engineers train policies on differentiable physics.

Key features at a glance:

  • Contact modeling: Path and surface contacts with tunable friction.
  • Actuation: Motors, transmissions and tendon-driven systems.
  • Sensors/effectors: Position, velocity and force feedback loops.
  • Scalability: GPU acceleration via MJX for ML workloads.

MuJoCo equips builders to prototype intelligent systems where physics fidelity drives success.

Builder's Take Start with the `simulate` viewer or Colab notebooks to prototype models quickly—master XML for custom rigs before deep C++ tweaks. Expect a learning curve on contact tuning for stability; leverage Python bindings for ML pipelines unless raw performance demands the C core. Check the changelog for version-specific solver changes before upgrading.

Bun Packs JavaScript Runtime, Bundler and Tools into One Executable

Written in Zig with JavaScriptCore, it replaces Node.js while slashing startup times and memory use for TypeScript developers.

oven-sh/bun Zig Latest: bun-v1.3.10 87.8k stars

Developers building JavaScript and TypeScript applications have a new all-in-one option: Bun, a single executable that combines a high-performance runtime, bundler, test runner, and package manager. Launched in 2021 by oven-sh, Bun targets the pain points of fragmented Node.js workflows, where separate tools like npm, Webpack, and Jest slow down development.

At its heart is the Bun runtime, engineered as a drop-in Node.js replacement. Written in Zig—a low-level language emphasizing safety and speed—it leverages JavaScriptCore, Apple's engine from WebKit, rather than V8. This choice yields dramatic gains: startup times under 1ms in benchmarks and memory usage halved compared to Node.js for many workloads. Run bun run index.tsx and it handles TypeScript and JSX natively, no extra configuration needed.

Beyond the runtime, Bun embeds production-grade tools. Its package manager mimics npm but installs dependencies into a lean cache, avoiding bloated node_modules folders. bun install pulls packages compatibly with existing package.json files. The bundler supports React, TypeScript transpilation, and hot module replacement out of the box. Testing is straightforward: bun test executes Jest-compatible suites with 10x speedups reported in common scenarios.

Script execution mirrors Node: bun run start launches package.json scripts, while bunx cowsay 'Hello, world!' runs one-off binaries without global installs. All this ships in one ~50MB binary, installable via curl script on Linux (kernel 5.1+), macOS (x64/ARM), or Windows (x64/ARM64):

curl -fsSL https://bun.sh/install | bash

Upgrades are seamless: bun upgrade. The latest v1.3.10, released recently, includes refinements from 11 contributors like project lead @jarred-sumner and @cirospaciari. Full notes detail stability tweaks and platform support—check Bun's blog.

Bun shines for full-stack builders tired of tool sprawl. It integrates into CI/CD pipelines via Docker images (oven/bun) and supports canary builds for bleeding-edge testing (bun upgrade --canary). With 87,000+ GitHub stars signaling broad adoption, its technical edge lies in unifying the stack: faster iteration without rewriting projects.

For Node.js teams, migration is incremental—most APIs work unchanged. Edge cases, like certain native modules, may need polyfills, but Bun's compatibility layer handles 99% of npm ecosystem packages.

Builder's Take Test Bun on a greenfield TypeScript project first, verifying CPU compatibility (AVX2 on x64) to avoid "illegal instruction" errors. Existing Node apps can swap `node` for `bun` gradually, but audit native addons for JavaScriptCore quirks. Dive into the docs for runtime flags and bundler options before production deployment.

Open-Source DIY Analog Modular Synth Hits Eurorack Rails

Shmøergh Moduleur supplies schematics, PCBs and firmware for self-contained synthesizer modules

shmoergh/moduleur · HTML · 173 stars

The GitHub project shmoergh/moduleur delivers complete designs for a fully analog, DIY modular synthesizer system, derived from the Shmøergh Hog. Targeted at experienced builders, it publishes schematics, Gerbers, BOMs, panel files, firmware and mechanical designs for Eurorack-compatible modules.

Key modules include:

  • Power supply with ±12 V rails and reverse polarity protection
  • VCO with 1 V/oct control
  • Mixer plus sidechain compressor
  • VCF, ADSR & VCA
  • Analog bitcrusher, S&H, LFO and output stage
  • Brain digital controller with ongoing firmware updates

Modules operate standalone in a default patch or integrate into any Eurorack setup, emphasizing robust signal flow, available components and clear hackability. Simulation files in CircuitJS and test notes support modifications.

Now at v1.0.1 (9 months since June 2025 launch), recent updates fix PCB errata on VCA and Brain boards without functional changes. Hardware uses CC BY-NC 4.0; software MIT. A roadmap outlines module improvements; join Discord for collaboration.

This open repository enables reproducible analog instruments, lowering barriers for tinkerers seeking customizable audio hardware amid rising synth costs.

Builder's Take Review the build guide and fixlog before ordering PCBs; test with simulations for safe prototyping. Contribute firmware patches via pull requests to enhance the Brain module.
