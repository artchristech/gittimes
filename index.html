MIRA Simulates Rocket League Matches Inside a 5B Parameter World Model 🔗

Real-time 2v2 gameplay generated from player actions via latent diffusion on a single GPU

mira-wm/mira · Python · 333 stars 4d old · Latest: v1.0

MIRA is a research project that turns the idea of game simulation on its head: instead of rendering frames from a traditional game engine, it generates Rocket League gameplay in real time using a 5-billion-parameter latent diffusion model conditioned on the actions of all four players. The result is a fully interactive 2v2 match that runs inside the model at 20 FPS on a single consumer GPU, with no access to the original game engine or assets. Each frame is synthesized from a representation of the game state and player inputs, effectively making the neural network itself the world model.

The project, released just four days ago by a collaboration between General Intuition, Kyutai, and Epic Games, includes the model weights, training code, and a synchronized multi-view dataset of Rocket League gameplay. Every sample in the dataset captures a four-second window from all four players’ perspectives, paired with keyboard actions and game state (ball position, car dynamics, score). This allows MIRA to learn not just how the game looks, but how actions translate into outcomes across multiple viewpoints—a crucial step toward generalizable, interactive world models.

Installation is streamlined via pixi, with a one-time setup command that pulls dependencies and prepares the environment. The test suite validates core functionality, and a live demo is available at mira-wm.com. The model operates in a latent space where temporal consistency is enforced through autoregressive frame generation, enabling coherent, minutes-long simulations without drift—a notable achievement given the complexity of Rocket League’s physics and fast-paced interactions.

What makes MIRA technically compelling is its fusion of representation learning and generative modeling. Rather than treating each frame as an independent image generation task, it uses representation autoencoders to compress game states into a structured latent space, which the diffusion model then evolves step-by-step based on player actions. This decouples perception from dynamics, allowing the model to generalize to novel compositions of cars, ball trajectories, and player behaviors not explicitly seen in training.

For developers, MIRA opens a path toward agent training in safe, controllable environments where the cost of failure is zero. It also suggests a future where game AI doesn’t rely on hand-coded rules or expensive physics simulators, but learns emergent behavior from data alone. Researchers in reinforcement learning, robotics, and embodied AI may find value in studying how MIRA bridges perception, action, and prediction in a closed-loop interactive setting.

The catch: MIRA is currently scoped to Rocket League’s 2v2 mode only, and while it generates visually plausible gameplay, the long-term stability of generated matches beyond short clips and the model’s ability to handle out-of-distribution actions remain untested questions.

Engram turns Claude Code into a self-testing tutor for lasting learning 🔗

The plugin adds spaced repetition and free-recall checks to AI explanations, ensuring knowledge sticks beyond the chat session

nagisanzenin/engram · Python · 455 stars 4d old

Engram transforms Claude Code from an explainer into a learning engine by inserting a verification loop between query and answer. After a user invokes /learn kalman filters, the system does not immediately provide an explanation. Instead, it first prompts the learner to produce an answer from memory, then reveals the canonical response only after confidence is assessed via a picker interface—no typed numbers allowed.

This reversal, detailed in release notes v0.4.4, corrects an earlier flaw where a stale oath line overrode the new anti-sycophancy protocol, ensuring the tutor now reliably gathers self-assessment before unveiling the target concept.

Under the hood, Engram employs the FSRS (Free Spaced Repetition Scheduler) algorithm to schedule reviews. Each successfully recalled concept earns a future /review date, visualized in the README as a growing chain of dots representing 2–4 minute sessions. The system stores all state locally in ~/.claude/learning/ as plain JSON files—no accounts, no cloud sync, no subscription. Three slash commands govern the flow: /learn initiates a new topic, /review triggers recall practice, and /coach offers metacognitive guidance. A quiet session hook alerts users only when reviews are due, minimizing distraction.

The project’s omni-repo design means the same engine works in OpenAI Codex via codex plugin marketplace add nagisanzenin/engram, appealing to builders who oscillate between environments. Early adoption shows traction in technical domains like Rust lifetimes and Kalman filters, where free-recall verification combats the illusion of competence that passive explanation often fosters.

The catch: Engram’s reliance on self-generated answers assumes learners possess sufficient foundational knowledge to attempt recall; for completely novel topics with no prior schema, the initial /learn prompt may yield blank responses, potentially undermining confidence before any explanation is given—a trade-off between active retrieval efficacy and novice accessibility that remains unaddressed in the current v0.4.4 release.

Use Cases

  • Data scientists retaining Kalman filter derivations
  • Developers mastering API nuances via scheduled review

The open source AI agent landscape is rapidly converging on a modular skill-based architecture, where discrete, reusable capabilities replace monolithic agents. This shift enables developers to compose complex workflows by snapping together specialized skills—much like installing plugins—rather than building agents from scratch. Evidence abounds in recent projects: alirezarezvani/claude-skills offers 345 pre-built skills spanning engineering, compliance, and finance, designed to plug into Claude Code, Cursor, and Gemini CLI.

Similarly, modiqo/skillspec (Rust) provides a framework to define, test, and prove agent skills with formal contracts and alignment proofs, ensuring reliability when chaining capabilities.

This modularity extends across domains. For research, mvanhorn/last30days-skill aggregates insights from Reddit, X, YouTube, and Polymarket into synthesized summaries, while xbtlin/ai-berkshire deploys multi-agent adversarial analysis using value investing frameworks from Buffett and Munger. In creative workflows, calesthio/OpenMontage delivers 500+ agent skills for video production—covering editing, montage, and audio—turning coding assistants into full studios. The catch: While the skill ecosystem is expanding rapidly, fragmentation looms—skills often lock into specific agent runtimes (e.g., Claude Code vs. Codex), reuse across platforms remains inconsistent, and validation of skill reliability in complex chains is still nascent. Many projects prioritize breadth over depth, raising questions about long-term maintainability and security, as highlighted by NVIDIA/SkillSpector’s focus on scanning for malicious patterns in skills—a sign that trust, not just availability, is the next hurdle.

Unlike the traditional C++ core, MJX runs simulations entirely on GPU or TPU hardware using JAX transformations, enabling vectorized rollouts and automatic differentiation without leaving the Python ecosystem.

The update streamlines workflows for researchers using MuJoCo in machine learning pipelines. Where prior versions required custom C++ extensions or slow Python loops for batch simulations, MJX now allows users to compile models once and execute thousands of parallel rollouts with jax.vmap and jax.grad.
