Brazilian Developers Build Unified Graph from Scattered Public Databases

Open-source infrastructure normalizes dozens of government datasets into a queryable Neo4j graph for civic analysis.

World-Open-Graph/br-acc Python Latest: v0.4.0 1.1k stars

Brazil's public data—spanning company registries, health records, education metrics, employment figures, public finances, procurement, and environmental reports—sits fragmented across dozens of government portals. Developers at World-Open-Graph have launched br-acc, an open-source Python project that ingests these datasets, normalizes them, and loads them into a single Neo4j graph database. The result: a queryable infrastructure surfacing entity relationships without interpretation or bias, empowering users to uncover connections themselves.

At its core, br-acc features 45 ETL pipeline modules, tracked in docs/source_registry_br_v1.csv with statuses like loaded, partial, stale, blocked, or not built. These pipelines handle official sources such as CNPJ (company registry), TSE (electoral data), DOU (official gazette), PGFN (debt collection), and Transparência (public spending). Data flows into a standardized Neo4j schema, enabling graph queries on normalized entities and links.

A React frontend provides search, corporate network exploration, and entity analysis tools. Developers access the graph programmatically via a FastAPI public API. Reproducibility is baked in: a one-command make bootstrap-demo spins up Docker services, Neo4j, the API, and seed data. Verify locally at http://localhost:8000/health (API), http://localhost:3000 (frontend), or http://localhost:7474 (Neo4j Browser). Privacy defaults align with LGPD, avoiding personal data exposure.

The latest v0.4.0 release bolsters reliability and collaboration:

  • Dual authentication: Bearer tokens plus httpOnly cookies with session restoration via GET /api/v1/auth/me.
  • Investigation sharing: POST /api/v1/investigations/{id}/share with expiration and revocation.
  • Enhanced search: Lucene-escaped queries with server-side counts.
  • ETL validation: Pandera schemas in etl/src/bracc_etl/schemas/ for key sources.
  • CI hardening: Strict mypy, ruff linting, automated audits; improved download scripts for CNPJ, DOU, STF, and more.

Rate limiting keys on authenticated users, not just IP, and CORS supports credentials. No API breaking changes ensure backward compatibility.

Launched two days ago with over 1,000 stars, br-acc stands out for builders tackling civic tech. Its modular ETLs serve as blueprints for similar graphs elsewhere, while the community bootstrap framework—new in v0.4.0—eases contributions via scripts, CI templates, and workflows. For developers eyeing public data projects, it models scalable, privacy-respecting ingestion at speed.

Builder's Take Start with `make bootstrap-demo` to test the stack locally before scaling ETLs. Review `docs/source_registry_br_v1.csv` to prioritize pipelines matching your data needs, and leverage Pandera schemas for robust validation. Expect Docker-heavy orchestration; ensure Neo4j familiarity, as custom queries drive value.

These enhancements make OpenClaw suitable for builders crafting custom workflows, from PDF analysis in chats to voice-enabled bots. With 251,501 stars signaling developer interest, it prioritizes single-user, privacy-first AI over enterprise scale.

Who cares? Solo developers, indie hackers, and teams valuing data sovereignty will find it a lightweight alternative to bloated SaaS AIs, enabling rapid prototyping of personal agents without infra overhead.

PythonRobotics Supplies Clear Python Codes for Core Robotics Algorithms

A decade-old repository covers localization, SLAM, path planning and control for developers building autonomous systems.

AtsushiSakai/PythonRobotics Python 28.8k stars

Developers tackling robotics face a steep curve: translating complex algorithms into working code. PythonRobotics, created in 2016 by Atsushi Sakai, cuts through that with a curated collection of Python implementations and an accompanying textbook. It focuses on practical, widely used algorithms, prioritizing readability over abstraction.

The repository spans key robotics domains. Localization includes Extended Kalman Filter (EKF), particle filters, and histogram filters, each with self-contained scripts that visualize robot pose estimation from sensor data. Mapping covers Gaussian grid maps, ray casting, lidar-to-grid conversion, k-means clustering, and rectangle fitting—essentials for building occupancy grids from raw scans.

SLAM (Simultaneous Localization and Mapping) features Iterative Closest Point (ICP) matching and FastSLAM 1.0, enabling robots to map unknown environments while tracking their position. Path planning dominates the collection: grid-based searches like Dijkstra, A*, D*, and D* Lite; model-free methods such as Dynamic Window Approach (DWA) and Potential Fields; sampling-based planners including RRT, RRT*, Probabilistic Roadmaps (PRM), and State Lattice Planning with biased polar or lane sampling.

Advanced tracking implements Stanley control, LQR for speed and steering, Model Predictive Control (MPC), and even Nonlinear MPC with C-GMRES. Arm navigation handles N-joint inverse kinematics with obstacle avoidance; aerial sections cover drone trajectory following and rocket-powered landing; bipedal planning uses inverted pendulum models.

What sets PythonRobotics apart? Minimal dependencies—mostly NumPy and Matplotlib—make it runnable out of the box. Scripts include animations for intuitive grasp of algorithm behavior, from RRT* tree growth to EKF covariance ellipses. Documentation links to detailed guides, and with nearly 28,000 stars, it has steady traction over 10 years, with pushes as recent as last month.

For builders in autonomous driving, drones, or manipulators, this isn't a black-box library. It's a learning tool and reference implementation, letting engineers prototype, debug, and customize without reinventing wheels. Algorithms like LQR-RRT* or Quintic polynomial planning bridge theory to deployable code, accelerating development in resource-constrained environments.

Web Tool Visualizes URDF Robot Models in Browser

Structured workflows streamline kinematic design, hardware specs and AI-assisted auditing for simulations

OpenLegged/URDF-Studio · TypeScript · 222 stars

URDF-Studio, a TypeScript-based web application, provides a graphical interface for building and editing Unified Robot Description Format (URDF) models used in robotics. It replaces manual XML editing with a 3D workspace powered by Three.js, enabling precise manipulation of links, joints and geometries.

The platform organizes design into three modes:

  • Skeleton Mode for defining kinematic chains and topologies.
  • Detail Mode for visual and collision geometries via primitives or mesh imports like STL and OBJ.
  • Hardware Mode for electromechanical parameters, including motor selection and transmission ratios.

Users can assemble multi-robot models by merging URDFs with automatic joint re-parenting. A precision collision editor separates collision bodies from visuals for accurate physics. Real-time analytics display joint axes, centers of mass and inertia tensors, with partial reloading for fast iterations.

AI features generate structures from natural language prompts and perform six-category audits on physical plausibility, kinematics and naming. Models export directly to MuJoCo for simulation.

Launched in December 2025, URDF-Studio targets roboticists seeking simulation-ready designs without desktop software dependencies.
