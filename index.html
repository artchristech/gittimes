Figma, once a collaborative powerhouse, has increasingly walled off automation. Its MCP server from June 2025 offered read-only access, while tools like figma-use—launched January 2026—enabled full read/write via Chrome DevTools Protocol (CDP). Figma countered in version 126.1.2 by stripping --remote-debugging-port flags, crippling such integrations. Proprietary .fig formats and binary lock-in exacerbate supply chain risks: a vendor update can shatter workflows overnight.

Enter OpenPencil: an MIT-licensed, Figma-compatible editor that opens .fig files natively, supports copy-paste between apps, and runs fully local—no accounts, subscriptions, or internet required. At ~7MB, it's lightweight and AI-native, featuring built-in chat with tool use. Users supply their own API keys for models like those from OpenAI, avoiding lock-in.

The v0.4.2 release, pushed March 2, sharpens Figma interoperability:

  • Fixes clipboard paste by skipping non-visual nodes (variables, widgets, stickies, connectors); resolves text rendering via proper letterSpacing conversion from {value, units} objects to pixels; adds undo/redo entries; centers pasted content; triggers auto-layout recompute.
  • Improvements import layoutAlignSelf, clipsContent, fontWeight, italic, letterSpacing, and lineHeight.
  • Tests expand to 14 unit tests for clipboard import.

Currently porting 29 of 118 figma-use tools for AI agent capabilities, OpenPencil eyes multi-file tabs, expanded AI providers (Anthropic, Claude, Gemini, Ollama), code signing, deeper .fig compatibility, and CI integrations like design linting and visual regression.

This project arrives as design-to-code pipelines demand programmable tools. Like VS Code unlocked editors and LLMs reshaped code gen, OpenPencil positions designers and devs to own their stack. With active development but pre-production status, its momentum signals a potential shift from Figma's moat.

Hwloc Maps Node Topology for HPC, Endures with Steady Maintenance

Open-MPI's hardware locality library delivers portable discovery of cores, caches, and NUMA—essential for performance tuning in parallel systems.

open-mpi/hwloc 683 stars C Latest: hwloc-2.13.0

In the world of high-performance computing, where milliseconds dictate success, understanding your hardware's layout is non-negotiable. Enter hwloc (open-mpi/hwloc), a C library and suite of command-line tools that reveals the hierarchical topology of compute nodes. Created in September 2013, it has amassed 683 GitHub stars over 12.4 years, with minimal recent growth signaling its status as a mature, niche staple rather than a trendy newcomer. The most recent star arrived on February 26, 2026, and the repo saw a push as late as March 2, underscoring ongoing stewardship.

Hwloc excels at interrogating key elements: NUMA memory nodes, shared caches, processor packages, dies, cores, logical processors (threads), and even I/O devices. It collects attributes like cache sizes and memory details, presenting them via a portable C API or tools like lstopo, which generates visual topology diagrams. This locality intelligence is vital for HPC applications optimizing thread affinity, data placement, and load balancing—preventing costly cross-NUMA traffic or cache thrashing.

Portability defines hwloc's edge. It leverages standard OS queries, supporting:

  • Linux (cgroups, cpusets, memory targets on Intel Xeon Phi, ScaleMP vSMP, NumaConnect),
  • Solaris (processor sets, logical domains),
  • AIX,
  • Darwin/OS X,
  • FreeBSD/NetBSD variants (x86 CPUID fallback),
  • Windows.

Processor-agnostic where possible (x86, PowerPC, etc.), it sidesteps vendor lock-in. While aimed at HPC, any developer chasing code or data locality on multi-core systems benefits—think NUMA-aware allocators or MPI launchers.

Hwloc integrates deeply with ecosystems like Open-MPI, embedding topology awareness into distributed jobs. Its v2.13.0 release, the latest milestone, refines these capabilities amid evolving hardware like multi-die CPUs.

Note: Official tarballs live outside GitHub at open-mpi.org/projects/hwloc, with full docs there too. The repo serves as a code mirror, not a primary distribution point.

For builders, hwloc's endurance—low stars but persistent updates—affirms reliability in production HPC stacks, where flash-in-the-pan tools falter.
