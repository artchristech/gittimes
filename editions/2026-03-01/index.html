Last updated February 2026, it analyzes Wi-Fi Channel State Information (CSI) to detect movement—without cameras, IR sensors, or batteries—integrating natively with Home Assistant via ESPHome.

At its core, ESPectre sniffs Wi-Fi packets from nearby routers, extracting CSI data that reveals signal distortions caused by human motion. Subcarriers in the Wi-Fi spectrum shift subtly as bodies disrupt multipath propagation, enabling presence detection up to several meters. No line-of-sight needed; it works through walls.

Version 2.5 introduces an ML Detector: an on-device neural network that skips traditional calibration entirely. Experimental but promising, it runs locally on ESP32 with a snapshot build (-ml assets). Feedback is solicited in discussion #84. For custom setups, see the setup guide.

Recent release notes highlight production hardening:

  • ESP32-C5/C6 fix for multi-antenna routers using STBC: Handles doubled HT-LTF fields (256-byte packets), extracting valid 64-subcarrier estimates.
  • ESP32-C3 calibration stability: Resolves OOM crashes in Micro-ESPectre NBVI mode, now succeeding with 59 KB free heap.
  • 3x faster packet collection on C3 (28 → 80 pps) via optimized math (skipped sqrts, integer loops).

Hardware is minimal: ESP32 (C3/C5/C6 supported), passive Wi-Fi monitor mode. Place near a 2.4 GHz router for best results—docs include a sensor placement guide. Architecture splits into Micro-ESPectre (firmware) and host processing, with a two-platform strategy for flexibility.

Privacy stands out: No images or audio captured; data stays local. Security via GPLv3 license. Active CI, 100% codecov, and ongoing evolution (e.g., git_ref substitutions) signal reliability for HA automations like lights or alerts.

The GitHub project OpenC3/cosmos, a Ruby-based platform for command, control, and telemetry of embedded systems, has released v7.0.0-rc2. Launched in July 2022 with 206 stars, it supports topics including automation, hardware integration, IoT, and spacecraft testing. Last updated March 1, 2026, it handles TCP/IP, UDP, and serial protocols.

Core functionality covers telemetry display, graphing, operational scripting, command sending, logging, and log playback. Users configure "targets" for hardware like embedded devices, cell phones, or home automation setups.

The RC2 builds on v6 with a QuestDB-powered TSDB for 10x faster data retrieval, eliminating reducer microservices. Changes since RC1 include SCOPE-prefixed TSDB table names (requiring openc3.sh cleanup on upgrade, which drops data), TEMPLATE_BASE64 keyword, new admin themes, and fixes for TSDB, STRUCTURE keyword, Python microservices, and XTCE.

License shifts from AGPL to the COSMOS Builder's License. Not for production; testing advised. Badges show passing Playwright, Ruby/Python unit, and API tests, with Codecov coverage.

Documentation at openc3.com.

Builder's Take Developers testing embedded hardware should clone the repo, run `openc3.sh start` after config, and enable QuestDB console for SQL queries. Upgrade from RC1 requires cleanup to avoid table conflicts.
