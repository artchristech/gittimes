Astryx Unlocks Design System Flexibility with Open Component Internals 🔗

Lets developers compose, customize, and eject components while supporting human and AI workflows

facebook/astryx · TypeScript · 5k stars 5mo old · Latest: v0.1.2

Why this leads today Facebook’s ASTRYX introduces a TypeScript-based design system built for AI agent integration, offering a new approach to customizable UI development that could reshape frontend workflows.

Facebook’s Astryx design system, now in public beta, is gaining traction for its radical openness in a space often dominated by locked-down, opinionated toolchains. Built over eight years inside Meta and powering more than 13,000 internal applications, Astryx ships as a cohesive system of 150+ accessible React components, brand-level theming, dark mode, and a CLI — but its real distinction lies in how it treats component accessibility and customization.

Unlike most design systems that encapsulate components behind rigid APIs, Astryx exports its internal building blocks directly.

Developers can compose buttons, inputs, or cards from primitive elements at any level of abstraction. When deeper control is needed, the swizzle eject command pulls a component’s full source into the project, granting ownership without forking the entire library. This approach supports both rapid prototyping and long-term maintenance, allowing teams to adapt components to precise brand or interaction requirements without fighting the system.

Styling flexibility is another cornerstone. Astryx authors its styles with StyleX, but this implementation detail remains invisible to consumers. Teams can override styles using their existing tooling — whether Tailwind CSS, CSS modules, or plain className — without adopting a new styling paradigm. Theming works through CSS custom property overrides, enabling designers to shift colors, spacing, or radii globally without wrapping components or maintaining forks.

Notably, Astryx is engineered for dual use: its API, documentation, and CLI are structured so that both human developers and AI assistants interact with it identically. This alignment reduces friction in agent-assisted workflows, where code generation and editing rely on predictable, well-documented interfaces. Recent updates reinforce this focus: the latest release adds interruptible button behavior for async actions, a UMD bundle for direct CDN usage, and sticky column support in tables — all while addressing SSR hydration issues in hover cards.

The project’s momentum is evident: 5,001 stars, 290 forks, and 205 open issues indicate active community engagement, with maintainers pushing updates as recently as yesterday.

The catch: Astryx remains in beta, and while its open-internals model offers flexibility, it places greater responsibility on developers to manage component stability and upgrades — especially when ejecting or relying on internal APIs that may evolve before v1.0.

Use Cases
  • Enterprise teams building brand-aligned UIs with shared component primitives
  • Design systems engineers customizing themes without wrapper components
  • Developer-AI collaboration using consistent, predictable tooling for codegen

Running State-of-the-Art LLMs Locally Without Cloud Dependencies 🔗

A builder's guide to hardware, software, and tuning for private, high-performance AI inference

jamesob/local-llm · Shell · 464 stars 0d old

Jamesob’s local-llm repository offers a detailed, battle-tested blueprint for running state-of-the-art large language models entirely on-premises, targeting builders who want to escape API rate limits, data privacy concerns, or vendor lock-in. Rather than theoretical advice, it documents a working system built around four NVIDIA RTX Pro 6000 GPUs, providing 384GB of combined VRAM, paired with a cost-optimized last-gen EPYC platform sourced from eBay to keep base system costs around $5,600.

The guide dives into niche but critical hardware tuning: using a c-payne PCIe switch to enable peer-to-peer GPU communication, adjusting BIOS settings for bifurcation and ASPM, disabling ACS to keep traffic within the switch fabric, and applying kernel parameters like iommu=off to prevent NCCL hangs.

Power limiting is also addressed — a necessity when running $46k of silicon on a standard 110V circuit. The result, measured via the included measure-gpu-speed.sh tool, achieves Gen4 line rates of 27.5/50.4 GB/s with sub-microsecond latency.

Software-wise, the runners/ directory provides ready-to-run Docker Compose configurations for serving models like GLM-5.2-594B via vLLM, with support for advanced techniques such as DCP4+MTP5, delivering approximately 80 tokens per second at 460k context lengths. A separate runners/stt folder includes a Whisper-large-v3-based speech-to-text pipeline. All configurations are designed for immediate use, assuming the hardware matches the guide’s specifications.

The catch: The guide assumes access to high-end, hard-to-find enterprise GPUs and a willingness to engage in low-level system tuning — making it impractical for most developers without significant hardware budgets or kernel-level debugging tolerance.

5.1 release. Key updates include restoring lost symbols like =, <, and > in code examples and adding color highlighting for strings in BBCode. Number highlighting in BBCode was also corrected, and a missing comma in Lesson 23’s code example was added. The app remains a browser-based tool for absolute beginners to learn Godot’s GDScript, with desktop versions available via Itch.io for better performance. Despite steady community use—2,702 stars and 225 forksthe project shows signs of aging: its last commit was one day ago, but 128 open issues persist, indicating ongoing maintenance demands. The scope is intentionally narrow, focusing only on foundational GDScript concepts as a stepping stone to broader Godot development, not a full programming curriculum.
The catch: The app’s limited depth means learners must quickly transition to other resources to build real games, leaving a gap between basic syntax and practical application.

Use Cases
  • Beginners learn GDScript syntax in-browser

  • Hobbyists prototype Godot logic without setup
