Developers tired of proprietary AI coding tools now have a robust open-source alternative: OpenCode, a TypeScript-based AI coding agent designed for seamless integration into local workflows.

At its core, OpenCode acts as an interactive agent that processes natural language instructions to edit codebases, manage Git operations, and handle complex tasks. It excels in maintaining session persistence across Git worktrees and orphan branches—a common pain point fixed in the latest v1.2.27 release. The agent uses an effects-based architecture, refactoring services like QuestionService to track user queries reliably, even when aborted or interrupted.

Installation is developer-friendly, prioritizing simplicity with a YOLO-style curl command: 

curl -fsSL https://opencode.ai/install | bash

Package managers follow suit: brew install anomalyco/tap/opencode for macOS/Linux, scoop install opencode for Windows, or npm i -g opencode-ai@latest. Custom paths respect $OPENCODE_INSTALL_DIR or XDG standards, landing binaries in $HOME/bin or similar. A beta desktop app extends this to GUI users, available as DMG, EXE, or AppImage, with Homebrew cask support.

Recent updates underscore maturity. Core fixes include VCS watcher logic, inline branded ID schemas, and bumped chunk timeouts to 5 minutes for heavier tasks. Compaction messages now flag as agent-initiated, aiding audit trails. UI tweaks handle multiline pastes and prevent filename overlaps. Community patches, like @michaeldwan's worktree fix (#16389), highlight collaborative momentum.

What sets OpenCode apart? Its VCS-native design—cleaning pending entries, effectifying permissions, and ditching legacy handlers—ensures reliability in real repos. No cloud dependency; run it locally with your LLM of choice. Builders gain transparency over black-box agents, customizing for private codebases or air-gapped setups.

For Git-heavy workflows, OpenCode matters: it bridges AI smarts with version control without session loss or bloat. At 11 months old, with over 120,000 stars signaling interest, it's poised for broader adoption amid rising demand for sovereign dev tools.

This runtime bridges web UIs and terminals, streamlining data pipelines for builders.

eProsima Fast DDS Delivers Reliable Pub-Sub for Real-Time Distributed Systems

C++ implementation of OMG's DDS and RTPS standards enables configurable, scalable communications over unreliable networks like UDP.

eProsima/Fast-DDS C++ Latest: v3.5.0.0 2.8k stars

In the world of distributed real-time systems, reliable data exchange over unreliable transports is a persistent challenge. eProsima Fast DDS addresses this head-on as a mature C++ implementation of the Object Management Group's (OMG) Data Distribution Service (DDS) standard. By natively supporting the RTPS (Real-Time Publish-Subscribe) protocol, it ensures interoperability and pub-sub communications via UDP, with options for best-effort or reliable quality-of-service (QoS) policies.

What sets Fast DDS apart is its dual-API approach. The high-level Publisher-Subscriber layer prioritizes usability for standard DDS workflows, while the low-level Writer-Reader API grants direct access to RTPS internals. This flexibility suits builders needing fine-grained control. Key features include plug-and-play discovery—new nodes auto-join networks—modular scalability for heterogeneous devices, and interchangeable transports for optimized deployments.

Plug-and-play connectivity means applications discover each other dynamically, ideal for robotics and IoT. Configurable network behaviors allow tailoring protocols to specific I/O channels, supporting growth from simple sensors to complex clusters.

Fast DDS powers critical ecosystems. It's the default middleware for ROS 2 (Robotic Operating System) in all long-term support (LTS) releases and most others, underpinning autonomous robots and simulations. In EU R&D, it's integrated into FIWARE for smart city and robotics projects via the FIWARE Catalogue.

The project's 11.8-year evolution shows in its slow-burn reliability, with 2,752 GitHub stars reflecting steady adoption. The latest v3.5.0 release (March 2026) delivers targeted enhancements:

  • API extensions: DomainParticipant::get_instance_handle() now returns by value; added return codes for better error handling.
  • Discovery improvements: Super-client behavior for Discovery Server clients; enhanced filtering for virtual/external endpoints.
  • Refactors: RPC overhaul, Secure Discovery Server rework, removal of unused keep-alive logic.
  • Build/CI updates: Qt 6.8.3 integration, uncrustify formatting across sources, Fast DDS Gen 4.3.0 for type regeneration.
  • Fixes: PDPTests compilation, history QoS consistency enforcement.

These changes bolster stability for production use. Builders in robotics, distributed systems, or middleware should evaluate it for RTPS-native performance. Commercial support is available via info@eprosima.com.

For developers tackling latency-sensitive pub-sub, Fast DDS offers proven, standards-compliant tools without vendor lock-in.

Whisper.cpp Ports OpenAI Speech Model to Lightweight C/C++ Inference

Enables high-performance, offline speech-to-text on edge devices from iPhones to Raspberry Pi

ggml-org/whisper.cpp C++ Latest: v1.8.3 47.6k stars

Builders seeking to embed automatic speech recognition (ASR) in resource-constrained apps now have a production-ready option: whisper.cpp, a pure C/C++ port of OpenAI's Whisper model. Launched in late 2022, it delivers the full capabilities of Whisper's transformer-based ASR—transcribing speech in multiple languages with near-state-of-the-art accuracy—without Python, PyTorch, or runtime dependencies.

The core innovation lies in its ggml tensor library, confining the model to just whisper.h and whisper.cpp. This enables zero memory allocations at runtime, mixed F16/F32 precision, and integer quantization for efficiency. On Apple Silicon, it leverages ARM NEON, Accelerate, Metal, and Core ML for GPU acceleration, running fully offline on iPhone 13 as demonstrated in project videos. x86 gets AVX intrinsics; POWER uses VSX. Broader support spans Vulkan for NVIDIA GPUs, OpenVINO, Ascend NPU, and Moore Threads.

Platform portability is unmatched: MacOS (Intel/Arm), iOS, Android, Linux/FreeBSD, WebAssembly, Windows (MSVC/MinGW), Raspberry Pi, and Docker. Voice Activity Detection (VAD) via Silero v6.2.0 filters noise, with recent fixes for buffer overflows and Hebrew WASM support.

The v1.8.3 release, a maintenance update atop latest ggml, includes a 12x performance boost on integrated graphics, separate VAD/ASR in Ruby bindings, and Go binding enhancements for segmented processing. Tools like vad-speech-segments and server mode (no_context option) aid integration.

Getting started is straightforward:

git clone https://github.com/ggml-org/whisper.cpp.git
cd whisper.cpp
sh ./models/download-ggml-model.sh base.en

Compile and run inference on audio files, yielding real-time transcription.

For builders, whisper.cpp solves the deployment puzzle: Whisper's quality without cloud latency or bloat. Embed it in voice assistants, as shown in command-line demos, or custom apps. With 47k GitHub stars signaling adoption, it prioritizes edge inference where Python ports falter.

Who cares? Mobile devs targeting offline transcription; IoT engineers on Pis; cross-platform tools needing ASR without deps.

Go-Ethereum Provides Core Ethereum Protocol Client

Geth CLI enables full node operation across Ethereum mainnet, testnets and private networks

ethereum/go-ethereum · Go · 50.9k stars

The ethereum/go-ethereum project, known as Geth, is the official Go-language implementation of the Ethereum protocol's execution layer. Launched in late 2013, it serves as the primary entry point for interacting with Ethereum networks, supporting full nodes (default), archive nodes (retaining full historical state) and light nodes (live data retrieval).

Key executables in the cmd directory include:

  • geth: Main CLI client exposing JSON-RPC endpoints over HTTP, WebSocket or IPC for network access.
  • clef: Standalone account signing tool integrable with geth.
  • devp2p: Networking utilities for peer interaction without full blockchain sync.
  • abigen: Generates Go bindings from Ethereum ABI contracts.

Building requires Go 1.23+ and a C compiler; make geth produces the binary, with automated downloads at geth.ethereum.org.

The v1.17.1 release ("Eezo Shunt") fixes snap-sync regressions, security issues and p2p crashes. New features encompass --metrics.influxdb.interval for customizable reporting, inspect-trie subcommand for state trie analysis, and Amsterdam fork support including EIP-7834 (SLOTNUM opcode), EIP-8024 and EIP-7928 fixes. Networking drops eth/68 protocol and optimizes transaction/blob pool relaying to cut traffic.

OpenC3 COSMOS Provides Full Command Control for Embedded Hardware Targets

Ruby toolkit delivers telemetry monitoring, scripting, and UI over TCP/IP or serial for testing any connected device

OpenC3/cosmos Ruby Latest: v7.0.0-rc3 210 stars

OpenC3 COSMOS tackles a core challenge for builders: bridging software to physical hardware. This Ruby-based system acts as a command, control, and communication layer for "targets"—any embedded device reachable via TCP/IP, UDP, serial, or similar protocols. It supplies out-of-the-box tools to send commands, receive telemetry, log data, and automate operations, eliminating the need to cobble together disparate scripts and interfaces.

At its heart is the Command and Telemetry Server, which manages target connections and displays raw packet data. Builders configure interfaces once, then access a suite of applications: the Limits Monitor tracks telemetry points exceeding thresholds, logging violations even after recovery; the Command Sender offers dropdowns for one-off packets with parameter descriptions; and the Script Runner executes Ruby test procedures line-by-line in a graphical debugger, with pause/stop controls and telemetry checks.

For visualization, COSMOS includes telemetry displays, graphing, and log playback. These enable rapid iteration on hardware prototypes, from verifying cell phones to orchestrating IoT setups. Scripts handle operational flows or tests, making it ideal for regression suites on embedded systems.

Version 7.0.0-rc3, the latest release candidate, introduces a QuestDB time-series database (TSDB) as the headline upgrade. This replaces slower storage, boosting data retrieval by an order of magnitude and dropping reducer microservices. SQL queries via the QuestDB console unlock new analytics, while fixes address streaming APIs for tools like Data Extractor, Data Viewer, and Telemetry Grapher. Python UV installation and security enhancements round out changes. A pre-update openc3.sh cleanup is required, which drops logged data.

The project shifted from AGPL to the COSMOS Builder's License, easing commercial use for integrators. Active since 2022 with 210 GitHub stars, it draws from aerospace roots but applies broadly. Builders in hardware testing, IoT, or space benefit from its protocol-agnostic design—no vendor lock-in, just plug in your target and script.

QuestDB's speed positions COSMOS for scale: imagine querying years of telemetry in seconds for anomaly detection. For production, await final 7.0; RCs suit exploration. This isn't just a tool—it's infrastructure for hardware-software symbiosis.
