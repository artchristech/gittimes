PythonRobotics Delivers Clean Python Codes for Essential Robotics Algorithms

A decade-old repository provides readable implementations and textbook-style explanations for localization, mapping, and path planning in autonomous systems.

AtsushiSakai/PythonRobotics Python 28.8k stars

Developers building robots, drones, or autonomous vehicles often grapple with opaque algorithms for navigation and control. PythonRobotics, created by Atsushi Sakai in 2016, addresses this with a curated collection of Python codes that double as a textbook. It strips algorithms to their essentials, prioritizing clarity over complexity.

The repository spans core robotics challenges. In localization, it implements Extended Kalman Filter (EKF) for odometry and IMU fusion, particle filters for global estimation, and histogram filters for probabilistic positioning. Mapping covers Gaussian grid maps, ray casting, and LiDAR-to-grid conversion, plus k-means clustering for object detection and rectangle fitting for scene understanding.

SLAM gets practical treatment via Iterative Closest Point (ICP) matching and FastSLAM 1.0, enabling simultaneous mapping and localization without heavy frameworks. Path planning dominates with grid-based methods like Dijkstra, A, D, and D Lite; model-free approaches such as Dynamic Window Approach (DWA) and Potential Fields; and sampling-based planners including RRT, RRT (with Reeds-Shepp paths), LQR-RRT*, and Probabilistic Roadmaps (PRM). Advanced options like State Lattice Planning, biased polar sampling, and quintic polynomial trajectories support structured environments like highways.

Path tracking includes Stanley control, LQR for speed and steering, and nonlinear Model Predictive Control (MPC) using C-GMRES. Specialized navigation covers N-joint robotic arms with obstacle avoidance, drone 3D trajectories, rocket-powered landings, and bipedal walking via inverted pendulum models.

What sets PythonRobotics apart is its design: algorithms are easy to read for grasping core ideas, selected for practicality, and built with minimal dependencies—mostly NumPy and Matplotlib for visualization. Each module includes animations to illustrate behavior, making it ideal for prototyping. Documentation links to a full guide, a YouTube overview, and a research paper (arXiv:1808.10703).

Sustained updates—last push in early 2026—keep it relevant amid steady traction, including nearly 29,000 stars. For builders, it solves the implementation gap: no need to reinvent EKF or RRT from scratch. Instead, fork, tweak, and deploy in simulations or hardware like ROS-integrated robots.

This resource matters because robotics demands reliable, understandable code. PythonRobotics equips developers to tackle autonomous navigation without vendor lock-in or black-box libraries, accelerating projects in self-driving cars, warehouse bots, or aerial delivery.

Builder's Take Start with the documentation's "Getting Started" section and run examples in a virtual environment to verify dependencies like NumPy and SciPy. Focus on one algorithm family, like path planning, before branching out—each module is modular for easy integration into larger systems. Expect pure Python implementations suited for education and prototyping, not real-time production without optimization.

Librealsense Delivers Cross-Platform Depth Sensing for RealSense Cameras

Intel's open source SDK streams high-res depth and color data, enabling robotics, drones and 3D scanning on multiple platforms.

realsenseai/librealsense C++ Latest: v2.57.6 8.6k stars

Developers building computer vision applications have long relied on Intel RealSense cameras for robust depth perception. The librealsense project, now under the realsenseai organization, provides the core SDK 2.0—a C++ library that unlocks these cameras' potential across platforms.

At its heart, librealsense handles depth and color streaming alongside intrinsic and extrinsic calibration data. It leverages stereo vision in RealSense hardware to compute depth maps, performing reliably in varied lighting and environments. This makes it indispensable for tasks like robotics navigation, 3D scanning, drone obstacle avoidance, skeletal tracking, and facial authentication.

The library stands out for its extensive wrapper support, including Python, ROS, C#, and Unity, allowing builders to integrate RealSense without deep C++ expertise. Cross-platform compatibility spans Windows 10/11, Ubuntu 20.04/22.04/24.04 (kernels 5.x and 6.x), macOS (High Sierra 10.13.2+), Android (7+), NVIDIA Jetson (JetPack 5-7), and Docker containers.

Recent beta release v2.57.6 emphasizes forward momentum. Key updates include:

  • Enhanced macOS support with updated installation guides.
  • NVIDIA JetPack 7.1 compatibility for USB-based SKUs.
  • Early access firmware for the D555 camera, boosting ROS2 integration and multicamera stability.

Beta code now lands directly on the master branch, with binaries slated for public servers like APT. While this release offers source-only assets (PyPI beta via pyrealsense2-beta), the Releases page hosts the last stable v2.57.6. Full changelogs detail bug fixes across platforms.

After a decade of steady development—migrated from IntelRealSense to realsenseai—librealsense remains the de facto standard for 3D stereo in robotics. Its open source nature fosters a vibrant ecosystem, with 8,558 stars reflecting sustained developer interest.

For builders, this SDK solves the challenge of accessible, high-fidelity depth sensing without proprietary lock-in. Whether prototyping a drone or scaling robotic fleets, librealsense provides the calibrated streams and tools to turn camera data into actionable insight.
