Brazilian Public Data Graph Links Scattered Databases into Queryable Neo4j Network

Open-source br/acc project normalizes official registries for entity analysis, with v0.4.0 adding dual auth and community tooling.

World-Open-Graph/br-acc 897 stars Python Latest: v0.4.0

Developers building civic tools now have a production-ready foundation in br/acc, an open-source graph infrastructure from World-Open-Graph that unifies Brazil's fragmented public databases. Launched in March 2026, the Python-based repository has garnered 897 stars in under a year, reflecting demand for structured access to siloed government data.

At its core, br/acc ingests sources like company registries (CNPJ), public finances, procurement records, health metrics, education data, employment stats, and environmental reports. It runs 45 ETL pipeline modules—tracked in docs/source_registry_br_v1.csv with statuses like loaded/partial/stale/blocked/not_built—normalizing them into a Neo4j graph. This schema exposes entities and relationships for querying corporate networks, procurement links, and more, without interpreting or scoring data. Users surface raw connections to inform civic decisions.

The stack prioritizes usability and extensibility:

  • React frontend for search and network exploration at localhost:3000.
  • FastAPI public API for programmatic access, with health checks at /health.
  • Neo4j browser at localhost:7474 for direct graph inspection.
  • Reproducibility via one-command bootstrap: cp .env.example .env && make bootstrap-demo, spinning up Docker services with seed data.

Version v0.4.0, pushed days ago, hardens the project for collaboration. Key upgrades include dual authentication—Bearer tokens plus httpOnly cookies with session restoration via GET /api/v1/auth/me—and rate limiting per authenticated user. A new community bootstrap framework streamlines contributions with CI templates and scripts. Developers gain investigation sharing (POST /api/v1/investigations/{id}/share with expiration/revocation), Lucene-escaped search with server-side counts, and Pandera-validated ETL schemas for sources like TSE, DOU, and PGFN.

CI improvements enforce mypy strict, ruff linting, and automated audits, with inline type-ignore for edge libraries like pyarrow and defusedxml. No API breaking changes ensure backward compatibility, while CORS tweaks support cookie auth.

This matters because Brazil's open data, mandated by law, remains portal-locked and format-inconsistent. br/acc delivers a privacy-compliant (LGPD-aligned, no personal data) single source of truth, enabling builders to prototype analyses without custom scrapers. With full ingestion via make orchestration for BYO data, it's primed for forks tackling procurement fraud or environmental compliance graphs.

Recent activity—last push March 2, 2026—signals momentum, making it noteworthy for devs eyeing graph-powered public interest tools.

Builder's Take Clone and run `make bootstrap-demo` for a local demo with seed data before committing to full ETL. Review `docs/source_registry_br_v1.csv` to gauge pipeline maturity and prioritize contributions. Expect Docker-heavy workflows; bring your own data for production-scale ingestion, and leverage the new community bootstrap scripts for custom pipelines.

hwloc Delivers Portable Hardware Topology for HPC Workloads

Open MPI's hwloc project maps NUMA nodes, cores and caches across OSes to optimize parallel code locality.

open-mpi/hwloc 683 stars C Latest: hwloc-2.13.0

In the world of high-performance computing, understanding a system's hardware topology is crucial for squeezing out performance. The hwloc project from Open MPI, hosted at open-mpi/hwloc, provides developers with a C API and command-line tools to query and visualize these topologies precisely.

At its core, hwloc constructs a hierarchical map of a node's key elements: NUMA memory nodes, shared caches, processor packages, dies, cores, processing units (logical processors or "threads"), and even I/O devices. It gathers attributes like cache sizes and memory details, enabling applications to bind threads to optimal locations, minimize data movement, and exploit locality.

This matters because modern platforms—from multi-socket servers to accelerators—feature complex topologies that generic OS schedulers often mishandle. hwloc helps HPC workloads, such as MPI applications, but extends to any code targeting data or instruction locality. Builders tuning parallel algorithms on NUMA systems or heterogeneous hardware will find it indispensable.
