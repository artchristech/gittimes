The Git Times

API & MCP Documentation

Access GitTimes data programmatically via a REST API or MCP server (Model Context Protocol). Both expose the same data: edition archives, trending repos, star velocity, and coverage history.

Quick Start

MCP (for Claude Code, Cursor, and other MCP clients)

Add to your MCP client configuration:

{
  "mcpServers": {
    "gittimes": {
      "command": "node",
      "args": ["/path/to/gittimes/mcp-server.js"]
    }
  }
}

Your agent can then call tools like get_latest_edition, search_repos, and get_trending_repos directly.

REST API (for scripts, agents, and integrations)

# Start the server
node api-server.js

# Query the latest edition
curl http://localhost:3717/api/editions/latest

# Search for repos
curl "http://localhost:3717/api/repos/search?q=react"

MCP Server

The MCP server uses stdio transport and exposes 9 tools. It reads directly from the GitTimes SQLite database and generated HTML files. No network requests, no API keys required.

Available Tools

Tool Description
list_editionsRecent editions with headlines, dates, repo counts
get_editionFull edition detail by date (repos + article excerpts)
get_latest_editionMost recent edition with full details
search_reposFind repos featured in any edition by name
get_repo_historyStar/fork/issue snapshots over 14 days
get_repo_coverageHow many editions featured a repo
get_sectionsNewspaper sections with topics and languages
get_trending_reposRepos ranked by star velocity
get_edition_statsAggregate archive stats

REST API

Lightweight HTTP server (Node.js built-in http module, no frameworks). Binds to 127.0.0.1:3717 by default. Override with HOST and PORT environment variables.

All responses are JSON with Access-Control-Allow-Origin: * for browser/agent access.

Endpoint Reference

GET/api/editions
List recent editions with headlines, dates, and repo counts.
?limit=10
Number of editions (1-100, default 10)
GET/api/editions/latest
Full detail for the most recent edition, including all featured repos and article excerpts.
GET/api/editions/:date
Full detail for a specific edition by date.
:date
Edition date in YYYY-MM-DD format (e.g. 2026-03-27)
GET/api/repos/search
Search for repos featured across editions. Returns grouped by repo with edition appearances.
?q=react
Repo name or partial match (required, max 200 chars)
GET/api/repos/:owner/:name/history
Star, fork, and issue snapshots over the last 14 days. Includes computed star velocity.
:owner/:name
Full repo identifier (e.g. facebook/react)
GET/api/repos/:owner/:name/coverage
How many times a repo has been featured, with article headlines per edition.
:owner/:name
Full repo identifier
?lookback=30
Number of recent editions to check (1-100, default 30)
GET/api/sections
All newspaper sections with their topic tags, languages, and article budgets.
GET/api/trending
Repos with the highest star velocity between the two most recent snapshots.
?limit=15
Number of repos (1-50, default 15)
GET/api/stats
Aggregate archive statistics: total editions, unique repos, date range, snapshot counts.
